# RapidIdentity — Alternate Actions Input/Output Contracts

Alternate Actions replace or extend built-in Portal behavior. The `AA` prefix is required on all action sets used as alternate actions.

## Return Record Pattern

Every AA action set must return a JSON-serialized record. Use `toJSON(returnRecord)` as the final return value.

```xml
<action name="createRecord" outputVar="results"/>
<action name="setRecordFieldValue"><arg name="record" value="results"/><arg name="field" value="&quot;success&quot;"/><arg name="value" value="true"/></action>
<action name="setRecordFieldValue"><arg name="record" value="results"/><arg name="field" value="&quot;message&quot;"/><arg name="value" value="&quot;Operation successful.&quot;"/></action>
<action name="return"><arg name="value" value="toJSON(results)"/></action>
```

Minimum return fields: `success` (boolean), `message` (string). Additional fields per action documented below.

## Profiles Alternate Actions (replace built-in Portal logic)

### AAChangePassword

| Input Property | Notes |
|---|---|
| `target_dn` | DN of user whose password is changing |
| `perp_dn` | DN of the user performing the operation |
| `audit_event_id` | Audit correlation ID |
| `newPassword` | The new password |
| `currentPassword` | Only present when perp = target (self-service) |
| `mustUpdate` | Only present when perp ≠ target. `true` = force change at next login |

Return: `success`, `message`

---

### AAUpdateProfile

| Input Property | Notes |
|---|---|
| `target_dn` | DN of user being updated |
| `perp_dn` | DN of the performer |
| `audit_event_id` | Audit correlation ID |
| `record_string` | JSON object of LDAP attributes → new values. Cleared attributes have `null` value. Multi-valued = array. |

Return: `success`, `message`, optional `new_dn` (return if the object moved in the directory)

---

### AADisableAccount / AAEnableAccount / AAUnlockAccount

All three take the same inputs:

| Input Property | Notes |
|---|---|
| `target_dn` | DN of the account |
| `perp_dn` | DN of the performer |
| `audit_event_id` | Audit correlation ID |

Return: `success`, `message`

---

### AAClaimAccount

| Input Property | Notes |
|---|---|
| `target_dn` | DN of the account being claimed |
| `perp_ip` | IP address of the performer (IP, not DN) |
| `audit_event_id` | Audit correlation ID |
| `newPassword` | Temporary password generated by Portal — **must be set on the account** |

Return value is ignored by Portal. Still log success/failure internally.

## Groups Alternate Actions (extend built-in group logic — return value ignored)

### AACreateGroup

| Input Property | Notes |
|---|---|
| `group_dn` | DN of the newly created group |
| `perp_dn` | DN of the performer. May be `null` during import. |
| `audit_event_id` | Audit correlation ID |

---

### AAUpdateGroup

| Input Property | Notes |
|---|---|
| `group_dn` | DN of the group |
| `perp_dn` | DN of the performer. May be `null` during import. |
| `audit_event_id` | Audit correlation ID |
| `old_record` | JSON with pre-update state: `name`, `description`, `type`, `ownerDNs`, `coOwnerDNs`, custom attributes |

---

### AADeleteGroup

| Input Property | Notes |
|---|---|
| `group_dn` | DN of the deleted group |
| `perp_dn` | DN of the performer |
| `audit_event_id` | Audit correlation ID |
| `record_string` | JSON with pre-deletion group state (same shape as `old_record` above) |

---

### AASyncGroup

| Input Property | Notes |
|---|---|
| `group_dn` | DN of the synced group |
| `perp_dn` | DN of the performer. May be `null` for auto-sync. |
| `audit_event_id` | Audit correlation ID |

---

## Sponsorship Alternate Actions (replace built-in People module logic)

### AACreateSponsored

| Input Property | Notes |
|---|---|
| `perp_dn` | DN of the sponsor creating the account |
| `audit_event_id` | Audit correlation ID |
| `record_string` | JSON object of LDAP attributes → values for the new account |

Return: `success`, `message`, **`new_dn`** (required on success — DN of created object), **`password`** (required on success — included in confirmation email to sponsor)

---

### AAEditSponsored

| Input Property | Notes |
|---|---|
| `target_dn` | DN of the account being edited |
| `perp_dn` | DN of the performer |
| `audit_event_id` | Audit correlation ID |
| `record_string` | JSON object with changed LDAP attributes. Cleared = `null`. |

Return: `success`, `message`, optional `new_dn`

---

### AATransferSponsored

| Input Property | Notes |
|---|---|
| `target_dn` | DN of the account being transferred |
| `perp_dn` | DN of the performer |
| `audit_event_id` | Audit correlation ID |
| `old_sponsor_dn` | DN of the previous sponsor |
| `new_sponsor_dn` | DN of the new sponsor |

Return: `success`, `message`, optional `new_dn`

---

### AAExpireSponsored

| Input Property | Notes |
|---|---|
| `target_dn` | DN of the account being expired |
| `perp_dn` | DN of the performer |
| `audit_event_id` | Audit correlation ID |
| `record_string` | JSON object with expiration date attribute and new value, e.g. `{"idautoPersonEndDate":"20110308191013Z"}` |

**Important:** Action set is expected to both disable the account AND set the expiration date — Portal does not do either on its own when this AA is configured.

Return: `success`, `message`

---

### AACertifySponsored

| Input Property | Notes |
|---|---|
| `target_dn` | DN of the account being certified |
| `perp_dn` | DN of the performer |
| `audit_event_id` | Audit correlation ID |
| `record_string` | JSON object with updated expiration date attribute and value |

Return: `success`, `message`
